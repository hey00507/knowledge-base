- **Filter Chain 동작 원리**: SecurityFilterChain이 어떤 순서로 실행되는지

- **토큰 저장 전략**: 클라이언트에서 안전한 토큰 보관

### 3. **실무 패턴 연구**
- **Refresh Token 전략**: 토큰 갱신 메커니즘
- **Multi-Device 로그인**: 여러 기기에서 동시 로그인 관리
- **토큰 블랙리스트**: 로그아웃된 토큰 무효화 방법
- **마이크로서비스 인증**: API Gateway에서의 인증 처리

### 4. **보안 모범 사례**
- **OWASP Top 10**: 웹 애플리케이션 보안 취약점
- **암호화 모범 사례**: 비밀번호 해싱, Salt 사용
- **보안 헤더**: CSRF, XSS 방지 헤더 설정
- **API 보안**: Rate Limiting, Input Validation

## 💡 **핵심 인사이트**

### 1. **기술적 학습**
- **JWT는 Stateless하지만, 보안을 위해서는 적절한 상태 관리가 필요**
- **Spring Security의 Filter Chain에서 실행 순서가 인증 성공의 핵심**
- **비밀번호 암호화는 단순 해싱이 아닌 Salt + 느린 알고리즘(BCrypt) 필수**

### 2. **실무적 통찰**  
- **개발 환경과 프로덕션 환경의 보안 설정은 반드시 분리해야 함**
- **API 보안은 인증만으로 끝나지 않고, 로깅/모니터링/Rate Limiting이 필수**
- **테스트 자동화는 보안 기능에서 특히 중요 (회귀 테스트)**

### 3. **코드 품질**
- **설정값은 하드코딩하지 말고 환경변수나 외부 설정 파일 활용**
- **예외 처리는 보안 정보 노출을 방지하면서도 디버깅 가능하게**
- **도메인 중심의 패키지 구조가 기술 중심보다 유지보수에 유리**
